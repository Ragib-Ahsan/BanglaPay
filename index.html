The regulatory catalyst: NPSB interoperability

Bangladesh Bank's October 2024 circular mandated full MFS interoperability via NPSB by January 2025. This broke down walled gardens, creating a unified rail that BanglaPay is built upon.

IIPS coming: The Inclusive Instant Payment System (launching Sep 2025) will enable real-time, cross-institution settlements – making bank-first routing even more powerful.

Lessons from India's UPI

PhonePe and Google Pay grew by providing a superior UX on top of interoperable rails. Key takeaways:

  • UX is king – simple, fast interface drives adoption
  • Aggregation wins – one app for all financial needs
  • Intelligence creates stickiness – insights drive repeat usage
  • Zero-MDR for merchants – rapid adoption

BanglaPay's core thesis: a non-competitive aggregator

We don't hold funds or compete with MFS. We partner with them to increase their transaction volume and user satisfaction. This cooperative stance is essential for regulatory alignment and rapid integration.

Technical Architecture

Built for scale, security, and interoperability

A microservices architecture that connects seamlessly with NPSB, IIPS, and MFS APIs.

System Layers

  • Application Layer: Mobile apps (iOS/Android) – unified interface
  • Routing Layer: Intelligent transaction router – selects lowest-cost rail
  • Interoperability Layer: BanglaQR, API connectors – universal acceptance
  • Payment Rails: NPSB (current), IIPS (future), MFS – execute transactions
  • Merchant Dashboard: POS integration, analytics, reconciliation
  • Security & Compliance: Encryption, OTP, Bangladesh Bank compliance

Key Integration Flows

  • Account Linking: OAuth with MFS/banks → secure token storage
  • NPSB P2P Transfer: Validate recipient via NPSB directory → route payment
  • BanglaQR Payment: Scan → identify merchant settlement preference → route via NPSB

Non-Functional Requirements

  • Security: TLS 1.3+, AES-256, biometric auth, PCI DSS
  • Performance: App load <2s, payment <5s, 99.9% uptime
  • Scalability: Support for 1M+ concurrent users, horizontal scaling
  • Data Localization: All data stored in Bangladesh per central bank rules

Non-custodial model: BanglaPay never holds user funds. We only route transactions via partner rails. This eliminates risk and simplifies compliance.

NPSB Integrated

Future‑Ready with IIPS

Built for Bangladesh's next‑gen payments

The Inclusive Instant Payment System (IIPS) launched by Bangladesh Bank in September 2025 will revolutionize the country's financial infrastructure. BanglaPay is designed to seamlessly integrate with IIPS from day one.

What is IIPS?

IIPS enables real‑time, cross‑institution payments connecting banks, MFS wallets, digital banks, and agent channels. It will make settlements instant and cheaper.

  • Instant transfers across all financial players
  • Lower transaction costs – moving towards a cashless society
  • Enhanced financial inclusion for rural populations

BanglaPay + IIPS

As a neutral aggregation layer, BanglaPay is perfectly positioned to leverage IIPS. Our architecture is built to connect to IIPS effortlessly, enabling even faster and cheaper transactions.

Faster, cheaper cross‑platform payments
Connect even more financial providers
New use cases: G2P, micro‑savings, and beyond

Just as UPI transformed India, IIPS is Bangladesh's leap into a modern, inclusive payment ecosystem. BanglaPay is your gateway to this new era.
